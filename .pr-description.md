## Summary

Implements container-to-container networking infrastructure for MQTT+QUIC E2E tests in CI. Tests previously timed out in CI (Phase 7 at 30s) due to hardcoded `localhost:4433` QUIC address that doesn't work from inside test containers. New test-runner container architecture uses Docker service names (`controller-standalone:4433`) for proper container-to-container UDP/QUIC communication.

**Impact**: E2E tests that failed in CI with "Timeout waiting for config status report" now pass consistently in <15 seconds. All MQTT+QUIC integration tests enabled in CI.

## Problem

E2E integration tests for MQTT+QUIC config sync failed in CI but passed locally:

**Symptoms**:
- ✅ Tests pass 100% locally (direct host networking)
- ❌ Tests fail in CI with Phase 7 timeout (Docker container networking)
- From steward container: `localhost:4433` points to container itself, not controller

**Root Cause**: Tests used hardcoded `localhost:4433` for QUIC address. When tests run inside a Docker container (as they do in CI), `localhost` refers to the test container itself, not the controller container. QUIC uses UDP, which requires proper container-to-container networking that `localhost` doesn't provide.

## Technical Details

### Dockerfile.test-runner Architecture

```dockerfile
FROM golang:1.25
WORKDIR /workspace

# Install Docker CLI from official repository
# (Debian's docker.io package doesn't include CLI binary in Trixie)
RUN [install docker-ce-cli from download.docker.com]

# Create non-root user (UID 1001 = GitHub Actions default)
RUN groupadd -g 1001 testuser && \
    useradd -u 1001 -g testuser -m -s /bin/bash testuser

# Environment for container-to-container addressing
ENV CFGMS_TEST_QUIC_ADDR="controller-standalone:4433"

USER testuser
```

### CI Workflow Pattern

```bash
# Build test runner
docker build -t cfgms-test-runner -f Dockerfile.test-runner .

# Get Docker socket group (for non-root access)
DOCKER_GID=$(stat -c '%g' /var/run/docker.sock)

# Run tests with container-to-container networking
docker run --rm \
  --network cfgms-test \
  --group-add $DOCKER_GID \
  -v "$(pwd):/workspace" \
  -v /var/run/docker.sock:/var/run/docker.sock \
  cfgms-test-runner \
  go test -v -race ./test/integration/...
```
