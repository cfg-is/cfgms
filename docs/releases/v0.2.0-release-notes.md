# CFGMS v0.2.0 Release Notes

**Release Date**: July 28, 2025  
**Release Type**: Alpha Release  
**Sprint Goal**: Complete critical core features and early multi-tenancy/automation development

## Overview

CFGMS v0.2.0 marks a significant milestone in configuration management system development, delivering core infrastructure for enterprise MSP operations. This release establishes the foundation for hierarchical configuration management, automated script execution, and comprehensive REST API access.

## Major Features

### 🔗 REST API Implementation (Story 1)
Complete external integration capabilities for MSP tools and third-party systems.

**Key Features:**
- Comprehensive REST API endpoints for all core operations
- OpenAPI 3.0 specification with auto-generated documentation
- Secure API key authentication with HTTPS
- Integration endpoints for steward management and configuration access

**API Endpoints:**
- `GET /api/v1/stewards` - List all registered stewards
- `GET /api/v1/stewards/{id}` - Get specific steward details
- `GET /api/v1/stewards/{id}/config/effective` - Get merged hierarchical configuration
- `POST /api/v1/stewards/{id}/config` - Update steward configuration
- Plus health check and system status endpoints

### 🏗️ Configuration Inheritance System (Story 2)
Hierarchical configuration management with transparent inheritance and source tracking.

**Key Features:**
- **4-Level Hierarchy**: MSP (Level 0) → Client (Level 1) → Group (Level 2) → Device (Level 3)
- **Declarative Merging**: Named resources replace entire blocks to prevent conflicts
- **Source Tracking**: Every configuration value includes origin and hierarchy level
- **REST Integration**: `/api/v1/stewards/{id}/config/effective` returns merged configs with metadata

**Technical Implementation:**
```go
type ConfigValue struct {
    Value  interface{}
    Source string  // "msp", "client", "group", "device"
    Level  int     // 0=msp, 1=client, 2=group, 3=device
}
```

### ⚡ Script Execution Framework (Story 3)
Cross-platform script execution with comprehensive security and audit capabilities.

**Key Features:**
- **Multi-Platform Support**: Windows (PowerShell, CMD), Unix (Bash, Zsh, Python)
- **Security Framework**: Configurable signing policies (none/optional/required)
- **Module Integration**: Implements ConfigState interface for idempotent operations
- **Execution Control**: Configurable timeouts, environment variables, working directories

**Module Structure:**
- **Get/Set Pattern**: Script modules follow standard module interface
- **State Tracking**: Execution state persistence and result comparison
- **Error Handling**: Comprehensive error reporting and process management

### 📊 Script Execution Audit & Management (Story 5)
Complete audit trail and management capabilities for script execution.

**Key Features:**
- **Execution Logging**: Full audit trail for every script execution
- **Performance Metrics**: Execution time, resource usage, success rates
- **Query System**: Flexible audit record querying with filtering
- **History Management**: Per-steward execution history with configurable retention

**Audit Capabilities:**
- Script source tracking with cryptographic hashes
- Execution results with stdout/stderr capture
- Performance metrics and resource tracking
- Failed execution analysis and retry capabilities

### 📚 Documentation & Polish (Story 6)
Comprehensive documentation and integration testing for production readiness.

**Key Features:**
- **Integration Testing**: End-to-end verification of all v0.2.0 features working together
- **User Guides**: Configuration inheritance and script module documentation
- **API Documentation**: Complete REST API guide with examples
- **Release Preparation**: Version tagging and deployment readiness

## Breaking Changes

### Module Interface
- Updated ConfigState interface for better field-level comparison
- Enhanced module registration for script execution support
- Modified error handling patterns for consistency

### Security Model
- Default API key generation (auto-generated if not configured)
- Enhanced certificate management for REST API endpoints
- Stricter validation for script execution policies

## Deployment Notes

### Requirements
- Go 1.23+ for deployment
- Certificate infrastructure for mTLS
- Database storage for audit logging
- Network access for REST API endpoints

### Migration Guide
- Existing configurations require update for inheritance support
- Module registrations need script module additions
- API clients should migrate to new REST endpoints

### Security Considerations
- Default API keys are auto-generated but should be replaced in production
- Script execution requires careful security policy configuration
- Certificate rotation procedures should be established

## Known Issues

### Pre-existing Service Tests
Some unit tests in `features/controller/service/config_service_test.go` have pre-existing failures unrelated to v0.2.0 features. These are documented as maintenance items for future releases and do not impact v0.2.0 functionality.

### Platform-Specific Considerations
- Windows script execution requires PowerShell 5.0+ or PowerShell Core
- Unix script execution requires bash, zsh, or python3 availability
- Script signing validation is OS-dependent and requires proper certificate infrastructure
