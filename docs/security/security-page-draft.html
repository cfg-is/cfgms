Reporting Security Vulnerabilities

⚠️ DO NOT report security issues publicly

Please do not open GitHub issues or discuss security vulnerabilities in public channels. This protects CFGMS users while we develop and deploy fixes.

If you believe you've found a security vulnerability in CFGMS, please report it to us privately:

Email: security@cfg.is

🔐 PGP Encrypted Email (Recommended)

For sensitive security reports, we strongly recommend using PGP encryption:

What to Include in Your Report

  • Type of issue - e.g., authentication bypass, SQL injection, XSS
  • Affected component(s) - controller, steward, API, specific feature
  • Attack scenario - Step-by-step reproduction instructions
  • Impact - What can an attacker achieve?
  • Affected versions - Which versions are vulnerable
  • Proof of concept - Code, screenshots, or logs
  • Suggested fix - If you have one (optional)

What to Expect After Reporting

  • Acknowledgment within 48 hours - We'll confirm receipt
  • Initial assessment within 5 business days - Severity and impact evaluation
  • Regular updates - We'll keep you informed during investigation
  • Credit - With your permission, we'll acknowledge you in advisories

Our Response Timeline

  • Critical vulnerabilities: Patch released within 7 days
  • High vulnerabilities: Patch released within 14 days
  • Medium vulnerabilities: Patch released within 30 days
  • Low vulnerabilities: Included in next regular release
Safe Harbor: We will not take legal action against researchers who responsibly disclose vulnerabilities according to this policy.
