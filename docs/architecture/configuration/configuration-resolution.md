# Configuration Resolution

## Overview

Configuration resolution is the process of determining which configurations apply to a specific endpoint and how they are merged to produce a final configuration. CFGMS uses a sophisticated resolution process that takes into account DNA matching, tenant inheritance, and configuration specificity.

For a high-level overview of the configuration management system, see [Configuration Management Overview](./overview.md).

## Configuration Structure

CFGMS organizes configurations in a hierarchical structure:

1. **Base Configurations** - Applied to all endpoints, inherited through the tenant hierarchy
2. **Group Configurations** - Applied to endpoints matching dynamic group criteria
3. **Endpoint Configurations** - Applied to specific endpoints, not inherited

## Dynamic Groups

Dynamic groups are defined using DNA attributes with simple or complex matching criteria:

```yaml
# Group Definitions
groups:
  # Simple group based on role
  web_servers:
    public: true  # Whether this group is inherited to child tenants
    match:
      dna.role: "web-server"
  
  # Complex group with multiple criteria
  high_performance_servers:
    public: true
    match:
      dna.cpu_cores: ">= 8"
      dna.memory_gb: ">= 32"
      dna.os: "ubuntu"
```

Groups can be marked as:

- **Public**: Inherited to child tenants
- **Private**: Not inherited, only applies within the defining tenant

## Group Inheritance

Groups are inherited through the tenant hierarchy:

1. Each tenant has a `groups.cfg` file defining its groups
2. Each tenant has an `inherited_groups.cfg` file containing groups inherited from parent tenants
3. When a child tenant is created or updated, its `inherited_groups.cfg` is generated by:
   - Reading the parent tenant's `groups.cfg`
   - Filtering to include only public groups
   - Merging with any existing inherited groups from the parent's `inherited_groups.cfg`

```go
type GroupManager struct {
    tenantMgr *TenantManager
    store     ConfigStorage
}

func (gm *GroupManager) UpdateInheritedGroups(tenantID string) error {
    // Get parent tenant
    parentID, err := gm.tenantMgr.GetParentTenant(tenantID)
    if err != nil {
        return fmt.Errorf("failed to get parent tenant: %w", err)
    }
    
    // Get parent's groups
    parentGroups, err := gm.store.GetGroups(parentID)
    if err != nil {
        return fmt.Errorf("failed to get parent groups: %w", err)
    }
    
    // Filter to public groups
    publicGroups := filterPublicGroups(parentGroups)
    
    // Get parent's inherited groups
    parentInherited, err := gm.store.GetInheritedGroups(parentID)
    if err != nil {
        return fmt.Errorf("failed to get parent inherited groups: %w", err)
    }
    
    // Merge groups
    mergedGroups := mergeGroups(publicGroups, parentInherited)
    
    // Save to inherited_groups.cfg
    return gm.store.SaveInheritedGroups(tenantID, mergedGroups)
}
```

## DNA-Based Matching

DNA (eg Deoxyribonucleic acid in living organisms) is a set of attributes that describe an endpoint, such as its role, environment, location, or any other metadata. DNA is used to match configurations to endpoints and determine group membership.

```go
type DNAMatcher struct {
    Attributes map[string]interface{}
}

func (m *DNAMatcher) Matches(dna DNA) bool {
    for key, value := range m.Attributes {
        if !matchDNAValue(dna[key], value) {
            return false
        }
    }
    return true
}

func matchDNAValue(actual, expected interface{}) bool {
    switch v := expected.(type) {
    case string:
        return actual == v
    case map[string]interface{}:
        if op, ok := v["operator"]; ok {
            switch op {
            case ">=":
                return actual >= v["value"]
            case "<=":
                return actual <= v["value"]
            case "contains":
                return strings.Contains(actual, v["value"])
            }
        }
    }
    return false
}
```

### DNA Matching Rules

1. **Exact Match** - All specified attributes must match
2. **Comparison Operators** - Configurations can use operators like `>=`, `<=`, `contains`
3. **Regex Match** - Configurations can use regular expressions for attribute values

## Configuration Resolution Process

The complete resolution process involves:

1. **Base Configuration Resolution** - Apply base configurations from the tenant hierarchy
2. **Group Resolution** - Determine which groups the endpoint belongs to
3. **Group Configuration Resolution** - Apply configurations targeting those groups
4. **Endpoint Configuration Resolution** - Apply endpoint-specific configurations
5. **Configuration Merging** - Merge configurations according to precedence rules
6. **Validation** - Validate the resulting configuration

```go
type ConfigResolver struct {
    tenantMgr *TenantManager
    groupMgr  *GroupManager
    store     ConfigStorage
}

func (r *ConfigResolver) ResolveConfig(endpoint *Endpoint) (*Config, error) {
    // 1. Get inheritance chain
    chain := r.tenantMgr.getInheritanceChain(endpoint.TenantID)
    
    // 2. Apply base configurations
    config := &Config{}
    for _, tenantID := range chain {
        baseConfig, err := r.store.GetBaseConfig(tenantID)
        if err != nil {
            return nil, fmt.Errorf("failed to get base config for tenant %s: %w", tenantID, err)
        }
        config = mergeConfigs(config, baseConfig)
    }
    
    // 3. Determine group membership
    groups, err := r.groupMgr.GetEndpointGroups(endpoint)
    if err != nil {
        return nil, fmt.Errorf("failed to get endpoint groups: %w", err)
    }
    
    // 4. Apply group configurations
    for _, group := range groups {
        groupConfig, err := r.store.GetGroupConfig(endpoint.TenantID, group)
        if err != nil {
            return nil, fmt.Errorf("failed to get config for group %s: %w", group, err)
        }
        config = mergeConfigs(config, groupConfig)
    }
    
    // 5. Apply endpoint configuration
    endpointConfig, err := r.store.GetEndpointConfig(endpoint.ID)
    if err != nil {
        return nil, fmt.Errorf("failed to get endpoint config: %w", err)
    }
    config = mergeConfigs(config, endpointConfig)
    
    return config, nil
}
```

## Configuration Merging

Configurations are merged according to a set of rules that determine how different configuration types are combined.

```go
func mergeConfigs(configs ...*Config) *Config {
    result := &Config{}
    
    for _, cfg := range configs {
        result = mergeSingle(result, cfg)
    }
    
    return result
}

func mergeSingle(base, override *Config) *Config {
    // Deep merge objects
    merged := make(map[string]interface{})
    
    // Copy base values
    for k, v := range base.Data {
        merged[k] = v
    }
    
    // Apply overrides
    for k, v := range override.Data {
        if v == nil {
            delete(merged, k)
        } else if baseObj, ok := merged[k].(map[string]interface{}); ok {
            if overrideObj, ok := v.(map[string]interface{}); ok {
                merged[k] = mergeMaps(baseObj, overrideObj)
                continue
            }
        }
        merged[k] = v
    }
    
    return &Config{Data: merged}
}
```

### Merging Rules

1. **Later Overrides Earlier** - Later configurations override earlier ones
2. **Arrays Are Replaced** - Arrays are replaced, not merged
3. **Objects Are Merged Recursively** - Objects are merged recursively
4. **Null Values Remove Properties** - Null values remove properties

## Configuration Specificity

Configurations are sorted by specificity to determine which ones take precedence during merging.

```go
type BySpecificity []*Config

func (a BySpecificity) Len() int { return len(a) }
func (a BySpecificity) Swap(i, j int) { a[i], a[j] = a[j], a[i] }
func (a BySpecificity) Less(i, j int) bool {
    // Calculate specificity scores
    scoreI := calculateSpecificity(a[i])
    scoreJ := calculateSpecificity(a[j])
    
    // Higher score = more specific
    return scoreI > scoreJ
}

func calculateSpecificity(cfg *Config) int {
    score := 0
    
    // Endpoint configs are most specific
    if cfg.Type == "endpoint" {
        score += 100
    }
    
    // Group configs are next most specific
    if cfg.Type == "group" {
        score += 50
    }
    
    // Base configs are least specific
    if cfg.Type == "base" {
        score += 10
    }
    
    // Deeper in tenant hierarchy = more specific
    score += cfg.TenantDepth * 5
    
    return score
}
```

### Specificity Factors

1. **Configuration Type** - Endpoint > Group > Base
2. **Tenant Depth** - Deeper in tenant hierarchy = more specific
3. **Group Specificity** - More specific group criteria = more specific

```yaml
# /cfgms/tenants/acme/base/base.yaml

      min_length: 12
      require_special: true
      require_numbers: true
    mfa:
      enabled: true
      methods:
        - type: "totp"
          issuer: "ACME Corp"
  
  monitoring:
    intervals:
      health_check: "5m"
      config_check: "15m"
    alerts:
      channels:
        - type: "email"
          address: "ops@acme.com"
```

### Group Configuration

```yaml
# /cfgms/tenants/acme/groups/web_servers.yaml
group:
  name: "web_servers"
  public: true
  match:
    dna.role: "web-server"
  config:
    packages:
      - name: nginx
        state: present
    services:
      - name: nginx
        state: running
        enabled: true
    files:
      - path: /etc/nginx/nginx.conf
        content: |
          # Nginx configuration
        mode: "0644"
        owner: nginx
        group: nginx
```

### Group Definitions

```yaml
# /cfgms/tenants/acme/groups.cfg
groups:
  web_servers:
    public: true
    match:
      dna.role: "web-server"
  
  database_servers:
    public: true
    match:
      dna.role: "database-server"
  
  internal_servers:
    public: false  # Private group, not inherited
    match:
      dna.environment: "internal"
```

### Inherited Groups

```yaml
# /cfgms/tenants/bravo/inherited_groups.cfg
# This file is automatically generated from parent tenant's groups.cfg

  web_servers:

      dna.role: "database-server"
```
