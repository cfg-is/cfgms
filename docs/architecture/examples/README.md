## Configuration Model

CFGMS uses a hierarchical configuration model with dynamic groups:

1. **Base Configurations** - Applied to all endpoints, inherited through the tenant hierarchy
2. **Group Configurations** - Applied to endpoints matching dynamic group criteria
3. **Endpoint Configurations** - Applied to specific endpoints, not inherited

### Dynamic Groups

Dynamic groups are defined using DNA attributes with simple or complex matching criteria:

```yaml
# Group Definitions
groups:
  # Simple group based on role
  web_servers:
    description: "Web server role"
    match:
      roles:
        - "web"
    settings:
      security:
        firewall:
          allowed_ports:
            - 80
            - 443
  
  # Complex group with multiple criteria
  high_performance_servers:
    description: "High performance servers"
    match:
      hardware:
        cpu_cores: ">= 8"
        memory_gb: ">= 32"
      os:
        family: "linux"
        distribution: "ubuntu"
    settings:
      monitoring:
        health_check:
          interval: 15s
```

### Group Inheritance

Groups are inherited through the tenant hierarchy:

1. Each tenant has a `groups.cfg` file defining its groups
2. Each tenant has an `inherited_groups.cfg` file containing groups inherited from parent tenants
3. When a child tenant is created or updated, its `inherited_groups.cfg` is generated by:
   - Reading the parent tenant's `groups.cfg`
   - Filtering to include only public groups
   - Merging with any existing inherited groups from the parent's `inherited_groups.cfg`

### Module Configuration

Module configurations are integrated directly into the main configuration files:

```yaml
# In base.cfg
modules:
  service:
    defaults:
      state: "running"
      enabled: true
    services:
      sshd:
        state: "running"
        config:
          port: 22
          permit_root_login: "no"

# In groups.cfg
groups:
  web_servers:
    match:
      roles:
        - "web"
    settings:
      modules:
        service:
          nginx:
            state: "running"
            config:
              worker_processes: "auto"

# In endpoint.cfg
modules:
  service:
    nginx:
      state: "running"
      config:
        worker_processes: 8
        worker_connections: 2048
```
